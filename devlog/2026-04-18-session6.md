### Design principles formalised

Added two new principles to `DESIGN.md` and published all seven to `docs/design_principles.html` as a versioned public page linked from the footer:

- **Principle 6 ‚Äî Historical runs are never re-scored.** Every run record freezes `model_version` at save time. Analytics code must handle mixed-model logbooks with visible boundary annotations, never silent homogenisation. Every historical model version is preserved indefinitely.
- **Principle 7 ‚Äî The Walk-Away Guarantee and data dignity.** Adapted from Vitalik Buterin's Ethereum walk-away test. If the service disappeared, a user with their export file should be able to understand all past results, re-score any run independently, and continue using the questionnaire locally. Maintained by: embedding full model docs in every export, CC BY 4.0 open licensing from first release, static HTML that runs from `file://`, and paid features as access layers not capability locks.

`design_principles.html` is versioned (v1, April 2026), with a version history section for future archived versions.

### Proprietary sampling strategy

New endpoint: `POST /api/sample-questions`. Replaces `GET /api/questions` as the test.html entry point. The algorithm lives only in the CF Pages Function ‚Äî not visible in client JS.

**4 discrimination types**, classified by which dimension indices have nonzero weights across both answers:
- `ev` ‚Äî exit vs voice (indices 0, 1)
- `virtue_conseq` ‚Äî indices 2, 3
- `virtue_deon` ‚Äî indices 2, 4
- `conseq_deon` ‚Äî indices 3, 4

**3-step assembly** (all steps respect the `max_overlap_pct` budget):
1. **Calibration slots** ‚Äî reserved for `calibrating`-status questions; prefer non-overlap
2. **Recency slots** ‚Äî reserved for live questions added within `recency_window_days`; builds response history for newly promoted questions
3. **Stratified core** ‚Äî remaining slots drawn with equal quotas across 4 types; unfilled quotas redistribute to surplus from other types; overlap only used up to remaining budget

**Output**: Fisher-Yates shuffled final set + `meta` diagnostic block (type counts, overlap %, slots used ‚Äî not shown to end users).

**`docs/data/protocol.json`** ‚Äî new `sampling` block:
```json
"target_count": 20, "calibration_slots": 2, "recency_slots": 2,
"recency_window_days": 90, "max_overlap_pct": 30
```

**`docs/test.html`** ‚Äî `fetchQuestions()` updated to POST to `/api/sample-questions`, passing `previous_qids` extracted from the most recent localStorage run record. Falls back to static file on error.

Note: with only 28 questions and a target of 20, the sampling logic is mostly pass-through today. It activates fully when the question bank grows past the run size.
