# Session 9 — 2026-04-20

## What was built

### Synperson focus group — full spec and demographic profiles

Designed and populated a 30-person synthetic focus group ("synpersons") for longitudinal personality test research. The system is fully specced and partially built; the remaining work (event histories, scripts, D1 migration) is in TODO.md.

**Concept:** Each synperson is a realistic fictional person with a fixed archetype. They take the Quadrantology test repeatedly over time, driven by an evolving life history. Their answers come from a Claude API call that receives their biography, a stochastically sampled subset of their event diary, and a summary of past results. Unlike real users, we have full access to all their answers and can steer their narratives. They eventually become public as Personal Circle profiles subscribers can add.

**Panel:** 5 per archetype × 6 archetypes = 30 synpersons. Global distribution: ~20 countries, 6 continents. Age range 29–61. Roughly gender-balanced. First languages include Arabic, Bulgarian, Czech, Gujarati, Igbo, Japanese, Korean, Norwegian, Punjabi, Serbian, Spanish, Tamil, Turkish, Twi, Ukrainian, Urdu, Yoruba, Zulu — the i18n roadmap item added this session is partly motivated by this panel.

**Files created:**
- `synpersons/_rig_schema.yaml` — format reference for all rig.yaml files
- `synpersons/_events_schema.md` — event format, date distribution, intensity scoring, ongoing generation rules
- `synpersons/_research_protocol.md` — logged vs QA run types, no-rescoring rule, editorial change logging protocol
- `synpersons/{30 directories}/rig.yaml` — full demographic profiles: birth date/city, nationality, ethnicity, gender, location, work, languages, education, income/net worth, political leaning, religion, health (height, weight, chronic conditions), and 4–6 named relationships each with archetype + stub + dynamic description

## Key design decisions

**Memory model:** Discounted-sampling approach for event memory. Retention probability `P(E) = min(1, (intensity/10)^α × exp(−λ × age_days))`. High-intensity events persist; low-intensity events fade. Hard floor: intensity ≥ 8 events always included. Tunable per-synperson via `memory_params` JSON. Future phases add recall-recency boost (working memory cache) and causal link prefetching.

**Results history:** Compressed into epoch summaries — baseline modal archetype, recent-window modal archetype, stress archetype (modal on runs following high-intensity events). These three lines replace a full run list in the prompt.

**Run types:** `logged` (permanent, archetype-evolving, ~weekly) vs `qa` (ephemeral, deterministic memory, unlimited). QA runs still call `/api/sample-questions` and contribute to the sequence log — this is intentional (they improve co-occurrence data just as real users do).

**No-rescoring rule:** Once a logged run is stored, its archetype and scores are canonical. Mirrors real-user policy. Rationale: the personality arc is only meaningful if each data point reflects what the model "believed" at the time.

**Events format:** Markdown files, AI-generated but human-editable. Events have narrative dates (YYYY-MM in the synperson's internal timeline), intensity scores, and first-person past-tense body text. Retcon events (`<!-- retcon: E{NNN} -->`) can revise interpretation of past events without altering the original text. Causal links are implicit now; a future engine infers them.

**Event date distribution:** Exponential density — more events in recent periods, fewer in formative years. But formative events are more intense (typically 7–8; occasional 9). For seed set of 15 events: 3 formative / 3 establishing / 4 rising / 3 recent / 2 fresh.

**New event intensity:** Log-normal distribution (μ=1.4, σ=0.55) for generated events; median ~4.1, mean ~4.7. Significant events (7+) are rare — ~15% of new events.

**Story evolution in real time:** Narrative dates track the real-world calendar. Events are populated to April 2026 at seeding. New events added between runs use the current real-world month. A synperson whose diary falls more than 3 months behind current date should be caught up in the next evolve pass.

**Architecture path:** Option A (local scripts) now, migrating to Option B (CF Worker endpoints + admin UI) once prompts are validated. Staged roadmap (Stages 0–6) covers the path from developer-only scripts to subscriber creation of their own synpersons.

**Relationships as test material:** Each rig.yaml includes 4–6 named relationships, each with an archetype, a stub personality, and a description of the dynamic with the synperson. These are the raw material for diary events — most events involve one or more of the named relationships. Cross-archetype pairings (e.g. Legalist with a Contrarian antagonist) create the structural tension that makes personalities evolve.
