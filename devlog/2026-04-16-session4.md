## Roadmap restructure

Analytics page formally deferred to Tier 2. `history.html` serves as the stub (log of results) for the foreseeable future. Decision rationale: the analytics page is the core product deliverable for the tracker framing, but building it well depends on a stable question bank, sampling logic, and subscription gating — all of which are earlier work. Better to defer and build it once than to build a chart on an unstable foundation.

STATUS.md now has a full four-tier structure:

- **Tier 1** — URL sharing + r.html, mobile UX, .ics, payload preview, entry points to circle/analysis
- **Tier 1.5** — copy/explainers, question bank expansion, question sampling logic, subscription state design, product portfolio UI, offline bundle
- **Tier 2** — analytics page (proper trendline), org admin UI
- **Tier 3** — Coach Mode
- **Tier 4** — LLM chat, x402/crypto, Google Drive
