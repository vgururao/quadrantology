## What was built

Converted the `gh-pages` branch from a frozen stable snapshot into an active marketing/preview site for quadrantology.com while V2 (the CF Pages build on master) is in development.

All work was on `gh-pages` only. Master is untouched.

**All 12 HTML pages**
- Removed Shop and How to Play from the nav. Both pages still exist at their direct URLs but aren't surfaced in navigation. Decision: keep them accessible for anyone who finds them, but don't present them as destinations while the site is in pre-launch state.

**`docs/data/v2-cta.html`** (new file)
- Single source of truth for the V2 sign-up CTA text and Google Form URL
- Fetched by both `index.html` and `test.html` via `fetch()` and injected into `#v2-cta-container`
- Edit this one file to update copy or swap the form link on both pages simultaneously

**`docs/quadrantology.css`**
- Added `.v2-teaser` styles (moved from inline `<style>` block on index.html to shared CSS)

## Key decisions

- **Shared fragment file for CTA**: Rather than duplicating the blurb in two places, a single `v2-cta.html` is fetched client-side by both pages. Simple and editable without touching page logic.
- **Intro screen before Q1, not a separate page**: Keeps the URL clean (`test.html`) and avoids a navigation step. The intro is short — two paragraphs — so it doesn't delay the user meaningfully.
- **Shop/How to Play unlinked, not deleted**: The pages aren't ready/relevant for the marketing site, but removing them could break any existing inbound links. Unlinking from nav is the safe middle ground.
- **gh-pages is now actively maintained**: It's no longer a frozen snapshot. Workflow: `git worktree add /tmp/qntlg-ghpages gh-pages`, edit, commit, push, `git worktree remove`.
